Problema de Negócio

“A empresa "Casa dos sonhos" atua com empréstimos, financiando o sonho da casa própria para pessoas físicas. A empresa tem uma grande área de atuação, desde a área urbana, semiurbano e a área rural. Como a empresa recebe diversos pedidos diários para o empréstimo e o processo de avaliação de empréstimo é longa, a empresa começou a perder possíveis clientes com a demora do processo ser finalizado, sendo assim a empresa deseja desenvolver um modelo automático para aprovação ou não do cliente de forma automática para ganhar tempo e eficiência.”

Como resolver esse problema com a ciência de dados

Entrando em campos técnicos, o problema que essa empresa apresentou é de classificação binária, no qual a solução é utilizar a base de dados da empresa que contém o histórico de aprovação / reprovação e diversas informações sobre cada caso individualmente. Essas informações serão utilizadas para montar um modelo de machine Learning que irá retornar com a resposta desejada.

Descrição do trabalho realizado

Para o desenvolvimento de um modelo de classificação é necessário primeiro entender a base de dados, esse entendimento abrange saber se os dados são qualitativos e/ou quantitativos, como é a distribuição desses dados, presença de vazios entre outras coisas. Essas informações são importantes para saber quais técnicas de feature engineering serão necessárias para os dados. Como esse é um modelo de classificação, uma das importâncias é verificar se os dados são desbalanceados, ou seja, uma das classes apresenta uma presença muito superior em quantidade do que outra, isso tende a gerar resultados ruins em modelos de ML, então técnicas de balanceamento são necessárias e serão testadas nesse modelo.

Sobre qual modelo de machine learning, não há como definir qual se sairá melhor, sendo a forma mais adequada testar os modelos principais e chegar a uma conclusão. Para a resolução do problema proposto foram testados 8 modelos de ML com diferentes tratamentos de dados que geraram gráficos de box para permitir a avaliação da dispersão dos erros.

Para avaliar o resultado dos modelos de ML a medida escolhida foi F1_SCORE, sendo uma média harmônica entre a precisão e o recall, sendo uma forma de equilibrar os 2 resultados.
